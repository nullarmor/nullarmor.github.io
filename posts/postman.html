Postman - Hack The Box

Resolution of the Postman machine

June 24, 2020 - 7 minute read -
HackTheBox

Postman is an easy level machine based on Linux, the initial approach is based on a CVE that exploits the Redis without authentication. The privilege escalation stage it’s necessary to do some enumerations in order to identify a system backup file to crack the classical Linux password format. The second stage of the privilege escalation stage is based on a web application exploitation.

Exploiting - Exploit adaptation

Starting with the web application located at port 1000 we have a web application named Webmin version 1.910, searching for any exploit on Google we got:

Webmin 1.910 - ‘Package Updates’ Remote Command Execution (Metasploit)

Unfortunately this exploit only works with an authenticated session, we need the credentials. Going on for the next port we noticed that there’s a Redis service running. Searching for any exploits that suits the version 4.0.9, the exploit works with the following logic:

  • Creates the RSA keys for SSH
  • Send them to Redis (without password)
  • Save the keys at authorized_keys with redis-cli command
  • Login through SSH without password
