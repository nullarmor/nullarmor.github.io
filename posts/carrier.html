Carrier - Hack The Box

Resolution of the Carrier machine

April 19, 2020 - 4 minute read -
HackTheBox

Carrier is a middle-level machine based on Linux, the main focus of the machine is the network related stuff. Since the user exploitation, we have a web application dedicated to monitor the network, then in the post exploitation we have a really challenging privilege escalation stage, going through pivoting to get last flag.

Recon

Starting with Nmap

The SNMP port is open, we can see if there’s some interesting strings from the device using the tool snmpwalk

On /docs, verifying the file that has a manual with errors, the error 450009 is somehow related to the SNMP string that we got earlier.

It’s possible to login into the web application using the login admin and the password NET_45JDX23.

Post Exploitation - Pivoting to another network

The privilege escalation is a bit challenging on this box, because it needs a solid network knowledge about router management, routes, etc.

Doing a small recon we can see that there’s a tool named vtysh, responsible for managing the network.

The image above was obtained during the recon process and it show the actual network map, with 3 different networks: AS100, AS200 and AS300.

Back to the recon on the web application, in the tickets we can see that there’s a ticket saying that the network 10.120.15.0/24 has a leak problem on the FTP service.

The information lead us to an attack named BGP Route Hijacking, where the attacker hijack the gateway between two networks, making him the Man In The Middle, sniffing the whole traffic.

More information about BHP Route Hijacking: BGP Route Hijacking by Akamai
