Beep - Hack The Box

Resolution of the Beep machine

June 24, 2020 - 6 minute read -
HackTheBox

Beep is an easy level machine based on Linux, with multiple ways to hack into the machine. The initial approach has 2 vulnerable services hidden into multiple services with open ports, difficulting the exploitation. The privilege escalation process is pretty easy, with multiple vulnerable binaries.

Recon

There’s too much services with open ports, making it confusing to us. After checking service by service, we noticed that most of them are not vulnerable, a classical rabbit hole. The service running on port 443 seems interesting because it’s a web application named Elastix.

Exploiting - LFI + RCE (1st method)

The web application vtiger CRM 5.1.0 has the following LFI vulnerability exploit available:

vTiger CRM 5.1.0 - Local File Inclusion

The vulnerability is present in the module named sortfieldjson:

https://localhost/vtigercrm/modules/com_vtiger_workflow/sortfieldsjson.php?module_name=../../../../../../../../etc/passwd%00

So now we’re able to read some system files. Googling about the web application config files that has some kind of critical information we got a documentation, in this documentation there’s an interesting file named manager.conf where the admin password is supposedly to be stored in plain text!

Verifying the existence of this file through our LFI vulnerability, the admin credentials is found!

These credentials works as well!

Using these credentials to login as root on SSH works and after that we are able to read the user and root flags!

Exploiting - LFI + RCE (2nd method)

To exploit the server using this method, we’re going to login on Elastix web application using the credentials found early:

Searching on Google for suitable exploits we found a potential RCE exploit:

FreePBX 2.10.0 / Elastix 2.2.0 - Remote Code Execution

After a couple tries trying to make it work, we noticed that the exploit has a wrong caller extesion for Asterisk Call Manager. Using a tool named svwar we successfully got the right call extension: 233

Edit the script and change the default call extension number to 233. Now executing the exploit and listening on port 4444, a reverse shell is spawned as asterisk user, letting us capture the first flag:

Exploiting - Shellshock

Googling about suitable exploits for the web application named Webmin, we found the following metasploit module:

Webmin 1.580 - ‘/file/show.cgi’ Remote Command Execution (Metasploit)

The exploit description says that it only works if we’re authenticated but it’s possible to exploit this server without any authentication reaching the /file/show.cgi file.

I made a custom shellshock exploit specifically designed for this box, this script uses the shellshock vulnerability and spawn an interactive shell us, it’s available on my GitHub :)
