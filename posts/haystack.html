Post Exploitation - Escalating privileges through multiple users

The escalate proccess in this machine is a bit painstaking for beginners because it requires a hard enumeration, patience and ELK knowledge.

Our next step is escalate the privileges to root user, to do it we need to enumerate more because now we have more privileges.

After enumerating some process we noticed that there’s a Logstash service running and doing a deeper recon we found the Logstash config directory located at /etc/logstash/config.d that contains 3 important files: filter.conf, input.conf and output.conf.

Translating: The Logstash will read, match the pattern Ejecutar\s*comando\s*:\sCOMMAND_TO_BE_EXECUTED and execute the COMMAND_TO_BE_EXECUTED on every file named logstash_a* under /opt/kibana directory.

Knowing this we can create a .sh file that contains our reverse shell!
