Jerry - Hack The Box

Resolution of the Jerry machine

April 19, 2020 - 2 minute read -
HackTheBox

Jerry is an easy level machine based on Windows, is heavily based on Tomcat and very easy in general, doesn’t require privilege escalation.

Recon

Starting with Nmap:

Accessing the port 8080 we found a web application running Apache Tomcat/7.0.88.

Clicking on Manager App it show us a HTTP authentication pop-up but failing at providing valid credentials results in a 403 page displaying the default login and password, tomcat:s3cret.

Then we got access to the application manager.

Exploiting

There’s a vulnerability in deploy’s area that let us upload a file (.war) containing a JSP webshell.

Using metasploit

A metasploit module it’s available, named Apache Tomcat Manager Authenticated Upload Code Execution

Using a custom exploit

I made a custom exploit to this, it’s a simple exploit that login into Tomcat and upload a JSP webshell, then executes a Powershell reverse shell payload after it. This script is available on my GitHub.

Jerry Exploit

Executing my exploit you can set your listening netcat and wait for the reverse shell session…

There’s no need to escalate any privileges because the Tomcat application is currently running as authority/system, so now we can read the user.txt and root.txt flag at the same time!
