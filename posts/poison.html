HackTheBox

Poison is a middle level machine based on FreeBSD with a very easy pwn-user stage, has three different ways to retrieve the initial shell, two using a LFI vulnerability and one with a simple base64 decode that just give us the SSH credentials.

Exploiting - Log Poisoning then RCE (2nd method)

It’s possible to execute system commands through LFI, more specifically using the technique named log poisoning. In order to exploit this we need to find a log file that execute PHP code, let’s try /var/log/httpd-access.log, the Apache log file. Every HTTP request is stored on this file so everything we need to do is change your HTTP Header with a malicious PHP command.

Exploiting - RCE through PHPInfo Poisoning (3rd method)

This method is more complex because it requires some specific scenario in order to work properly. This scenario includes a PHP Info page and a LFI entrypoint. It exploits a race condition on PHP temporary mechanism that allows the attacker to trigger the temporary file before the delete-process (less than 1 second).

More info:

PHP LFI to arbitratry code execution via rfc1867 file upload temporary files

LFI with PHPInfo() assistance
