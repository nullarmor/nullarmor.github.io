Exploiting - Heartbleed vulnerability

After playing with the 443 port for some time, we noticed that there’s a vulnerability named Heartbleed where the attacker is able to compromise the server by leaking the server information through a OpenSSL bug.

There’s a Metasploit module for this vulnerability located at auxiliary/scanner/ssl/openssl_heartbleed:

OpenSSL Heartbeat (Heartbleed) Information Leak

After running for multiple times without any relevant information, we finally got something in the middle out of this byte-mess. What caught our attention is the HTTP request made in the beginning of the memory leak, the HTTP request was sent to https://127.0.0.1/decode.php with an argument named text with the following content: aGVhcnRibGVlZGJlbGlldmV0aGVoeXBlCg==
