Celestial - Hack The Box

Resolution of the Celestial machine

April 19, 2020 - 3 minute read -
HackTheBox

Celestial is a medium level machine based on Linux where the initial exploit is harder than the post exploit stage. The initial exploitation relies on a Node vulnerability and the privilege escalation is pretty easy.

Recon

Starting with Nmap:

Accessing the port 3000 we can see that there’s a simple page doing a sum with two numbers and displaying the result on the page.

When inspecting the web application cookies we can see that there’s a base64 string encoded. Decoding it gave us the follow JSON

Exploiting - Exploiting a Node.js deserialization vulnerability

Searching about requests and responses in JSON format on Node.js we found a deserialization vulnerability where it’s possible to execute system command on the server (RCE)

Exploiting Node.js deserialization bug for Remote Code Execution (CVE-2017-5941)

The vulnerability is result of a module named node-serialize, when more specifically, in the function named unserialize that fail to validate the input supplied by the user, executing everything that will be inside a JSON value.

Encoding the following malicious payload in base64 and then sending it as profile cookie it’s possible to obtain a reverse shell.

Using the same logic I made a custom exploit in Python, it comes with an interactive shell so don’t worry about setting up listeners :) The script is available on my GitHub.

Celestial Exploit

Inside the server we can read the first flag and move to the next step.

Post Exploitation - Exploiting cron routines

Doing some enumeration we can see that there’s a file named output.txt on my home directory, the file content doesn’t makes sense but he has root permissions attached to him.

On ~/Documents it’s possible to see a file named script.py, the script just reflect his own output to output.txt file and has read/write permissions to us, allowing us to edit him with any content!

The script seems to be executed by root every 5 minutes through cronjob service.

Inserting a reverse shell payload inside the script and waiting 5 minutes gave us a root shell, allowing us to grab the last flag and finish the box.
